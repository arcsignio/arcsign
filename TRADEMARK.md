# ArcSign Trademark Policy

**Last updated:** 2026-05-13

## Status

"ArcSign", "ArcSign Pro", and the ArcSign logo are **unregistered
trademarks** of the ArcSign project (common law trademarks under
continuous, public use since 2025).

Registered trademark application (USPTO + EUIPO + TW + CN) is planned
at a future milestone (Pro NFT holders > 500). Until then, the
unregistered ™ status is asserted under common law.

## What you MAY do

- ✅ Refer to the project by name in articles, tutorials, social media.
- ✅ State that your product is "compatible with ArcSign" if it actually is.
- ✅ Use the name in academic / technical writing.
- ✅ Fork the source code under Apache 2.0.
- ✅ Build and distribute compiled binaries from unmodified source.
- ✅ Use screenshots / logos in journalism / educational content with
  attribution.

## What you MAY NOT do

- ❌ Distribute modified versions under the name "ArcSign" or any name
  containing "ArcSign", "Arc-Sign", or close variations like "ArcSign-X",
  "ArcSign Pro+", "ArcSign Lite", "ArcSign Plus", "ArcSign Community".
- ❌ Use the ArcSign logo for products / services other than ArcSign itself.
- ❌ Imply official endorsement, partnership, or sponsorship without
  written permission.
- ❌ Register domain names containing "arcsign" with intent to confuse
  (cybersquatting).
- ❌ Use the name in cryptocurrency tokens, NFT collections, or financial
  products.
- ❌ Use the name in app store listings for non-official forks.

## Fork naming rules

If you fork ArcSign and distribute the binary:

- ✅ Acceptable: `MyFork-Wallet`, `BlueWallet-Lite`, `CommunityVault`,
  `OpenVault`.
- ❌ Not acceptable: `ArcSign Pro+`, `ArcSign Lite`, `ArcSign Community`,
  `Arc-Sign Plus`, anything starting with "Arc" + "Sign".

When in doubt, change the name entirely. The Apache 2.0 license gives
you the code; the trademark policy keeps the name.

## Enforcement

We will issue cease-and-desist requests for clear violations,
particularly those that:

- Mislead users into thinking a fork is official.
- Direct users to non-official download channels (anything other than
  `dl.arcsign.io`).
- Risk user fund loss through impersonation.

## Verifying "official" ArcSign

Only these channels are official:

- **Website**: `arcsign.io` (and `dl.arcsign.io` for downloads).
- **GitHub**: `github.com/arcsignio/arcsign`.
- **Twitter / X**: [@arcsign](https://twitter.com/arcsign).
- **Discord**: invite link on the official website / README only.
- **Email**: `security@arcsign.io`, `hello@arcsign.io`.

Any other channel claiming to be ArcSign is not official.

## On-chain identity

Official ArcSign contracts and addresses are pinned at compile-time and
documented in [`OFFICIAL_ADDRESSES.md`](OFFICIAL_ADDRESSES.md).

If your binary uses different addresses, it is not official — see the
verification methods in that file.
